# 52 347 товаров с Wildberries: как мы собрали каталог конкурентов за 3 часа

В ноябре 2025 года к нам обратился владелец интернет-магазина электроники. Конкуренты на Wildberries демпинговали — он терял продажи, но не знал об этом до конца недели. Ручной мониторинг занимал у менеджера два часа в день, и то выборочно. Нужно было понять реальную картину: цены, остатки, спрос по трём категориям (смартфоны, наушники, планшеты).

Мы выгрузили 52 347 товаров за 3 часа. Клиент получил CSV с 15 полями данных по каждой позиции: артикул, цена, старая цена, скидка, рейтинг, количество отзывов, остатки на складах, бренд, категория, URL, ссылки на изображения, дата сбора, ID продавца, регион отгрузки, статус наличия. На выходе — очищенный файл, где цены приведены к единому формату, удалены дубликаты, отсеяны позиции без цен.

## Анатомия данных: 15 полей для анализа ниши

Клиент пришёл с запросом «нужны цены конкурентов». Мы расширили список — для полноценной аналитики 15 полей:

- **Название товара** — полная карточка с моделью, цветом, объёмом памяти.
- **Артикул** — уникальный идентификатор для поиска у поставщиков.
- **Бренд** — производитель, для оценки доли рынка.
- **Цена текущая** — реальная цена продажи на момент сбора.
- **Цена старая** — зачёркнутая цена, если есть скидка.
- **Скидка в процентах** — глубина дисконта.
- **Рейтинг** — средняя оценка от 1 до 5.
- **Количество отзывов** — косвенный показатель объёма продаж.
- **Остатки на складах** — количество доступных единиц по регионам.
- **Категория** — вложенность в каталоге (смартфоны → Android → Xiaomi).
- **URL карточки** — прямая ссылка на товар.
- **Ссылки на изображения** — массив URL фотографий товара.
- **ID продавца** — идентификатор магазина на Wildberries.
- **Регион отгрузки** — склад, с которого идёт отправка.
- **Статус наличия** — «в наличии», «заканчивается», «нет в наличии».

Этого достаточно для расчёта маржи, оценки конкуренции и принятия решения о завозе.

## Ручной труд против алгоритма: математика деградации

Менеджер открывает карточку товара, копирует цену в таблицу, переходит к следующей. Скорость — 30 секунд на позицию при идеальных условиях. 52 347 товаров × 30 секунд = 436 часов = 54 рабочих дня при 8-часовом графике.

К 40-му дню данные на первых страницах устареют. Wildberries меняет цены 3–5 раз в сутки на ходовые позиции. Менеджер физически не успеет собрать актуальную выгрузку — к моменту завершения работы первые 20% данных потребуют обновления.

Ошибки при ручном вводе — 2–5% по нашей статистике. Пропущенная цифра, неверный разряд, потеря знака — такие ошибки всплывают при анализе и требуют перепроверки.

Алгоритм работает иначе. Скрипт отправляет запрос к внутреннему API Wildberries, получает JSON-ответ, извлекает 15 полей, сохраняет в базу. Скорость — 200–300 товаров в минуту с учётом задержек. 52 347 позиций = 3 часа.

## Техническая реализация: как мы это сделали

Wildberries не хранит товары в HTML страницы. Данные подгружаются динамически через внутренний API при прокрутке каталога. Мы открыли DevTools, отфильтровали сетевые запросы, нашли эндпоинт `https://catalog-back.wildberries.ru/api/v1/search/catalog-data`.

Запрос принимает параметры: `page` (номер страницы), `cat` (ID категории), `sort` (сортировка). В ответ приходит JSON с массивом товаров — готовая структура для сохранения.

Сначала получили дерево категорий, нашли ID нужных разделов: смартфоны (12 456), наушники (8 932), планшеты (15 783). Написали скрипт на Python, который последовательно проходит по всем страницам каждой категории.

Wildberries блокирует при частых запросах с одного IP. После 100–150 запросов в минуту сервер возвращает 429 Too Many Requests или требует капчу. Мы использовали пул из 50 прокси-серверов. Скрипт чередовал их, делал 8–10 запросов через каждый IP, затем переключался. Задержка между запросами — 400–600 мс.

Три категории парсили параллельно — запустили три копии скрипта. Это ускорило сбор в три раза.

## Очистка данных: что пришлось выкинуть

Сырые данные содержат мусор. Мы применили фильтры:

- Удалено 1 847 дубликатов — одни и те же товары у разных продавцов.
- Отсеяно 3 214 позиций без цены — товары сняты с продажи или скрыты.
- Приведены к единому формату 12 500 значений — разные форматы цен (с пробелами, без пробелов, с валютой).
- Извлечены 89 432 ссылки на изображения — в среднем по 1,7 фото на товар.

Итоговый файл — 52 347 строк, 6,2 МБ в CSV. Клиент открыл в Excel без конвертаций.

## Что показал анализ

Средняя цена смартфона на Wildberries — 34 800 рублей. В сегменте до 15 000 рублей — 12% предложений. Спрос в этом сегменте высокий: количество отзывов на бюджетных моделях растёт быстрее, чем на флагманах.

Пять брендов (Apple, Samsung, Xiaomi, Realme, Tecno) занимают 68% каталога. Остальные 32% делят 200+ производителей. Конкурировать с топ-5 в лоб — демпинг до нуля. Ниша малоизвестных брендов даёт маржу 40–50%.

Товары с рейтингом выше 4,5 продаются на 18–22% дороже аналогов с рейтингом 3,5–4,0. Покупатели платят за отзывы.

Остатки по 15% позиций упали до 1–3 единиц на складах. Это либо высокий спрос, либо проблемы с поставками. Клиент отметил эти SKU для приоритетного завоза.

## Стоимость работы

Клиент сравнивал три варианта.

Ручной сбор: 54 рабочих дня, 80 000 рублей на зарплату менеджера. Данные устаревают в процессе сбора, ошибки 2–5%.

Готовая база: 40–60 тысяч рублей, выгрузка раз в месяц. Нет остатков, старых цен, ссылок на фото. Данные двухнедельной давности.

Парсинг под заказ: 3 часа, 18 000 рублей. Свежие данные на момент сбора, 15 полей, очистка включена.

Клиент выбрал третий вариант.

## Юридическая чистота

Парсинг публичных данных не нарушает закон РФ. Данные Wildberries доступны без логина и пароля — любой пользователь видит каталог. Мы не обходим платный доступ, не используем чужие аккаунты.

Нагрузка на сервера умеренная — задержки 400–600 мс между запросами, распределение по 50 прокси. Это сопоставимо с активным пользователем.

Копирование контента (фото, описания) для своего сайта — нарушение авторских прав. Мы собираем данные для аналитики, клиент не использует чужие изображения в своём магазине.

## Что делать дальше

Для разового анализа достаточно выгрузки в CSV. Для постоянного мониторинга настраиваем автоматический запуск по расписанию — раз в сутки или раз в неделю. Результаты сохраняем в базу данных или Google Таблицу.

Сравнение выгрузок показывает изменение цен, появление новых товаров, исчезновение позиций. Клиент видит демпинг конкурентов в день изменения, а не через неделю.

Чтобы получить такой отчёт, нужно определить категории, настроить парсер, запустить сбор, очистить данные. Мы берём это на себя — клиент получает готовый файл с комментарием по структуре.
